## Seed Script Notes

When writing `db/seed.ts`:

1. Use `db.insert(activities).values([...]).onConflictDoNothing()` to make seeding idempotent
2. Insert activities first, then memberships, then `family_memberships`, then `activity_memberships`
3. Do not seed `settings` (PIN hash and admin password hash are set manually via env vars — never seeded)
4. Coordinates are approximate — parent should verify and correct via admin panel
5. All seeded activities have `is_active = true`, `is_considering = false` (Initiative 2 adds these columns)
